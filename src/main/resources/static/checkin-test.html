CHECK-IN API

발급된 티켓을 실제 입장 처리까지 닫아보기

CHK-001 체크인 API를 빠르게 검증하는 테스트 페이지입니다. 결제 이후 발급된 qrToken을 입력해 POST /api/checkin을 호출하고, ISSUED -> USED 전환과 중복 체크인 방지를 바로 확인할 수 있습니다.

권장 확인 순서

  1. ADMIN Access Token을 저장합니다.
  2. qrToken을 GET /api/me/tickets 또는 결제 테스트 페이지에서 복사합니다.
  3. POST /api/checkin으로 체크인을 실행해 USED, usedAt을 확인합니다.
  4. 같은 qrToken으로 다시 호출해 중복 체크인이 막히는지 확인합니다.

STEP 1

QR 체크인

유효한 qrToken이면 티켓을 USED로 전환하고, 이미 사용한 티켓은 409 Conflict로 막습니다.

체크 포인트
  • 존재하지 않는 qrToken이면 404 Not Found가 내려옵니다.
  • 성공 시 응답과 티켓 조회 모두 status=USED, usedAt이 반영됩니다.
  • 같은 티켓을 다시 체크인하면 409 Conflict로 중복 입장을 막습니다.
