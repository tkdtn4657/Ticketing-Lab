PAYMENTS & TICKETS API

결제 확정과 내 티켓 조회를 한 화면에서 검증하기

PAY-001, PAY-002, PAY-003, TKT-001 흐름을 빠르게 확인하는 테스트 페이지입니다. 예약 생성 이후 POST /api/payments/confirm로 결제를 완료하고, 같은 화면에서 GET /api/me/tickets 응답까지 이어서 점검할 수 있습니다.

권장 확인 순서

  1. USER Access Token을 저장합니다.
  2. Reservation ID, Amount, Idempotency-Key를 입력합니다.
  3. POST /api/payments/confirm를 호출해 PAID 전환과 결제 응답을 확인합니다.
  4. GET /api/me/tickets로 발급된 좌석/구역 티켓 수와 식별자를 확인합니다.

STEP 0

USER 토큰

브라우저에서 결제 API와 티켓 API를 호출하려면 사용자 access token이 필요합니다.

TOKEN EMPTY

STEP 1

결제 확정

예약 금액과 멱등 키를 검증한 뒤 결제를 승인하고, reservation item 수량 기준으로 티켓을 발급합니다.

체크 포인트
  • 같은 Idempotency-Key로 같은 요청을 다시 보내면 동일한 payment 응답이 재사용됩니다.
  • amount가 reservation 총액과 다르면 409 Conflict가 내려옵니다.
  • 성공 시 reservation은 PAID, 결제는 APPROVED가 됩니다.
  • 구역형 item은 qty만큼 개별 ticket이 생성되고, holdQty -> soldQty로 이동합니다.

STEP 2

내 티켓 목록
