EVENTS API

이벤트 조회 API 테스트 콘솔

EVT-001 목록 조회와 EVT-002 상세+회차 조회를 한 화면에서 확인할 수 있는 테스트 페이지입니다. 상태 필터를 바꿔 목록을 가져오고, 원하는 이벤트를 선택해 상세 응답과 회차 배열을 바로 검증해보세요.

권장 확인 순서

  1. 목록 조회에서 status 필터를 선택하거나 비워둡니다.
  2. EVT-001 요청으로 현재 DB에 있는 이벤트 목록을 가져옵니다.
  3. 목록 카드의 상세 조회 버튼으로 eventId를 자동 입력합니다.
  4. EVT-002 응답에서 이벤트 정보와 shows[] 구조를 확인합니다.

EVT-001

이벤트 목록

GET /api/events

FILTER

상태 조건

status? = DRAFT | PUBLISHED | CANCELLED

EVT-002

이벤트 상세+회차

GET /api/events/{eventId}

STEP 1

이벤트 목록 조회

필터 없이 전체 조회하거나, 상태별로 이벤트를 좁혀서 확인할 수 있습니다.

응답 구조

{ "events": [...] }

필터를 비워두면 전체 목록을 조회합니다.

체크 포인트

status가 잘못되면 400 Bad Request

유효 상태값은 DRAFT, PUBLISHED, CANCELLED 입니다.

LIST

조회된 이벤트

No Events Loaded

STEP 2

이벤트 상세+회차 조회

목록 카드에서 바로 선택하거나 eventId를 직접 입력해 상세 응답을 확인합니다.

안내
  • 존재하지 않는 eventId를 조회하면 404 Not Found가 내려옵니다.
  • 회차가 없는 이벤트라면 shows 배열은 빈 배열로 응답됩니다.
  • 현재 애플리케이션 DB에 이벤트 데이터가 없다면 목록은 빈 상태로 표시됩니다.

DETAIL

선택된 이벤트 상세

No Event Selected

UTILITY

보조 작업

선택된 이벤트, 목록 카드, 상세 패널, 응답 로그를 처음 상태로 되돌립니다.

STATE

현재 상태

Ready
Filter ALL
Listed Count 0
Selected Event ID -
Selected Title -
