SHOWS API

회차 가용성 API 테스트 콘솔

SHW-001 응답에서 좌석 seats[]와 구역 sections[]가 어떻게 내려오는지 바로 검증할 수 있는 테스트 페이지입니다. showId를 입력하거나, 이벤트 테스트 페이지에서 이동해 현재 회차의 좌석 가용 여부와 구역 잔여 수량을 한 번에 확인해 보세요.

권장 확인 순서

  1. Show ID를 입력하거나 ?showId=1 형식으로 페이지를 엽니다.
  2. SHW-001 요청으로 좌석 목록과 구역 목록을 불러옵니다.
  3. available=true|false 값이 좌석 상태와 맞는지 확인합니다.
  4. remainingQty = capacity - soldQty - holdQty 계산이 기대값과 맞는지 확인합니다.

SHW-001

회차 가용성 조회

GET /api/shows/{showId}/availability

SEATS

지정 좌석

seatId, label, rowNo, colNo, price, available

SECTIONS

구역 잔여 수량

sectionId, name, price, remainingQty

STEP 1

회차 가용성 조회

회차 ID 하나로 좌석과 구역 가용 정보를 함께 조회합니다. 공개 조회 API라 토큰 없이 바로 테스트할 수 있습니다.

좌석 응답

seats[].available 로 가용 여부를 확인합니다.

응답 순서는 rowNo, colNo, seatId 기준으로 정렬됩니다.

구역 응답

sections[].remainingQty 로 잔여 수량을 확인합니다.

구역은 name, sectionId 순으로 정렬됩니다.
체크 포인트
  • 존재하지 않는 showId를 조회하면 404 Not Found가 내려옵니다.
    • `r`n
  • 쿼리 파라미터가 없으면 로컬 샘플 회차를 자동으로 찾아 먼저 조회합니다.
  • 좌석 인벤토리나 구역 인벤토리가 없으면 각각 빈 배열로 응답합니다.
  • 이 페이지는 좌석/구역의 현재 조회 상태를 보여주며, 상태 변경은 수행하지 않습니다.
