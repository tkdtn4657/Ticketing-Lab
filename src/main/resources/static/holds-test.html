HOLDS API

선점 생성부터 해제까지 한 화면에서 검증하기

HLD-001, HLD-002, HLD-003 흐름을 빠르게 확인하는 테스트 페이지입니다. 사용자 access token으로 홀드를 만들고, 같은 화면에서 상세 조회와 취소, 그리고 SHW-001 가용성 반영까지 함께 점검할 수 있습니다.

권장 확인 순서

  1. USER Access Token을 입력해 저장합니다.
  2. Show ID와 items JSON을 채워 POST /api/holds를 실행합니다.
  3. 생성된 holdId로 상세 조회를 호출해 상태와 item 가격을 확인합니다.
  4. 가용성 조회로 좌석 HELD, 구역 remainingQty 변화를 확인합니다.
  5. 홀드 취소 후 다시 가용성을 조회해 자원이 복구됐는지 확인합니다.

HLD-001

선점 생성

POST /api/holds

HLD-002

선점 조회

GET /api/holds/{holdId}

HLD-003

선점 취소

DELETE /api/holds/{holdId}

SHW-001

가용성 검증

GET /api/shows/{showId}/availability

STEP 0

USER 토큰

브라우저에서 /api/holds를 호출하려면 사용자 access token이 필요합니다.

TOKEN EMPTY

STEP 1

홀드 생성

좌석 홀드는 {"seatId": 1}, 구역 홀드는 {"sectionId": 1, "qty": 2} 형태로 보냅니다.

체크 포인트
  • 좌석과 구역 item을 하나의 요청에 함께 넣을 수 있습니다.
  • 같은 seatId 또는 sectionId를 중복으로 넣으면 409 Conflict가 내려옵니다.
  • 존재하지 않는 seatId/sectionId를 보내면 400 Bad Request가 내려옵니다.
  • 좌석은 홀드 후 available=false, 구역은 remainingQty 감소로 확인합니다.

STEP 2

홀드 상세 / 취소

생성 응답의 holdId를 그대로 사용하거나, URL에 ?holdId=...로 진입할 수 있습니다.
