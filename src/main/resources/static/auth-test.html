AUTH CONTROLLER

한 화면에서 인증 흐름 확인하기

Swagger가 API 문서를 보는 도구라면, 이 페이지는 signup -> login -> me -> refresh -> logout 흐름을 빠르게 직접 검증하기 위한 테스트 콘솔입니다.

권장 확인 순서

  1. 같은 이메일로 회원가입 후 userId 확인
  2. 같은 계정으로 로그인 후 access / refresh token 확인
  3. 내 정보 조회로 현재 사용자 정보 확인
  4. 토큰 재발급으로 refresh token 회전 확인
  5. 로그아웃 후 기존 refresh token 재사용 불가 확인

AUTH-001

Signup

POST /api/auth/signup

AUTH-002

Login

POST /api/auth/login

AUTH-005

Me

GET /api/auth/me

AUTH-003

Refresh

POST /api/auth/refresh

AUTH-004

Logout

POST /api/auth/logout

STEP 1

계정 정보

회원가입과 로그인은 같은 입력값을 공유합니다.

예상 결과

{"userId": number}

같은 이메일로 다시 요청하면 409 Conflict

예상 결과

{"accessToken", "refreshToken"}

응답 본문과 헤더 기준으로 토큰을 확인합니다

STEP 2

토큰 작업

로그인 성공 후 값이 자동으로 채워집니다.

체크 포인트
  • 중복 회원가입은 409 Conflict가 나와야 합니다.
  • Refresh 성공 후에는 이전 refresh token이 더 이상 유효하지 않아야 합니다.
  • Logout 이후에는 기존 refresh token으로 재발급이 실패해야 합니다.
