RESERVATIONS API

예약 생성부터 내 예약 목록까지 한 화면에서 검증하기

RES-001, RES-002, RES-003 흐름을 빠르게 확인하는 테스트 페이지입니다. 사용자 access token으로 hold를 예약으로 전환하고, 같은 화면에서 상세 조회와 내 예약 목록, 그리고 SHW-001 가용성 반영까지 함께 점검할 수 있습니다.

권장 확인 순서

  1. USER Access Token을 입력해 저장합니다.
  2. Hold ID로 POST /api/reservations를 호출해 예약을 생성합니다.
  3. 생성된 reservationId로 상세 조회를 호출해 상태와 총액, items를 확인합니다.
  4. 내 예약 목록으로 페이지/상태 필터 응답을 점검합니다.
  5. 가용성 조회로 좌석 RESERVED, 구역 remainingQty 반영을 확인합니다.

HLD-002

선점 확인

GET /api/holds/{holdId}

RES-001

예약 생성

POST /api/reservations

RES-002

예약 상세

GET /api/reservations/{reservationId}

RES-003

내 예약 목록

GET /api/me/reservations

STEP 0

USER 토큰

브라우저에서 예약 API를 호출하려면 사용자 access token이 필요합니다.

TOKEN EMPTY

STEP 1

예약 생성

활성 holdId 하나를 결제 대기 상태의 reservation으로 전환합니다. 만료되었거나 이미 전환된 hold는 실패합니다.

체크 포인트
  • 성공 시 hold는 CONVERTED, reservation은 PENDING_PAYMENT 상태가 됩니다.
  • 좌석은 RESERVED로 바뀌고, 구역 수량은 기존 hold 상태를 유지합니다.
  • 본인 hold가 아니거나 존재하지 않으면 404 Not Found가 내려옵니다.
  • 만료된 reservation 자원은 새 hold 생성 시 lazy release 됩니다.

STEP 2

예약 상세 / 내 목록

생성 응답의 reservationId를 상세 조회에 사용하고, 목록 조회는 페이지와 상태 필터를 함께 점검합니다.
